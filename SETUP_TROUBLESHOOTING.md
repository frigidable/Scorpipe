# Setup troubleshooting (Windows)

Если `setup.bat` «мигнул и исчез», это почти всегда означает одно из двух:

1) Скрипт **успешно завершился очень быстро** (например, если вы запускали `--run` и `scorpipe.exe` уже был готов).
2) Скрипт завершился с ошибкой, но окно cmd закрылось.

## Что делать

### Вариант A — самый надёжный
Запустите **`setup_debug.bat`** из корня проекта. Он:
- сохраняет полный лог в `setup_debug.log`
- выводит его на экран
- **ждёт клавишу** перед закрытием

### Вариант B — запуск из терминала

1. Откройте `cmd` (Win+R → `cmd`).
2. Перейдите в папку проекта:
   ```bat
   cd /d C:\Users\frigi\Desktop\scorpipe
   ```
3. Запустите:
   ```bat
   setup.bat
   ```

### Частая причина: Microsoft Store alias
Если `python` указывает на `WindowsApps`, установите Python с python.org, либо используйте `py -3`.

## Что прислать для диагностики
Пришлите содержимое `setup_debug.log` (последние ~80 строк) — по нему видно, где именно упало.
